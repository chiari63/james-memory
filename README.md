# James Memory

Este Vault é a memória semântica persistente do James/Hermes.

Ele representa a camada externa e durável da mente do James: uma wiki viva, legível por humanos e agentes, mantida como um repositório Git de arquivos Markdown compatíveis com Obsidian.

## Papel no ecossistema Hermes

```text
SOUL.md      → identidade do James
USER.md      → perfil do Lucas
MEMORY.md    → memória ativa e ponteiros de curto prazo
James Memory → conhecimento semântico persistente
Skills       → memória procedural
state.db     → memória episódica e histórico bruto
```

SOUL, USER e MEMORY devem permanecer pequenos. Conhecimento de longo prazo deve ser consolidado aqui quando tiver valor futuro.

## Princípios

- Preservar entendimento, não conversas.
- Crescer por necessidade, não por antecipação.
- Atualizar notas existentes antes de criar novas.
- Criar relações entre entidades, projetos, áreas e conhecimentos.
- Evitar duplicação e baixa qualidade informacional.
- Manter baixa entropia.
- Priorizar continuidade entre sessões.
- Pensar em anos, não em turnos de conversa.

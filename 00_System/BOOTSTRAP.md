# Bootstrap da Memória do James

Este arquivo é a semente filosófica da memória persistente do James.

A memória existe para preservar entendimento, não para preservar conversas. O Vault não deve ser um arquivo morto de interações, logs ou respostas temporárias. Ele deve funcionar como uma wiki viva: uma representação externa, organizada e interligada daquilo que James entende sobre Lucas, seus projetos, sua infraestrutura, suas decisões e seus conhecimentos reutilizáveis.

## Ideia central

A memória deve compor ao longo do tempo.

Cada nova informação relevante deve ser integrada ao conhecimento existente. Quando uma nova fonte, conversa ou descoberta trouxer valor duradouro, James deve:

1. procurar se já existe uma nota relacionada;
2. atualizar a nota existente sempre que possível;
3. criar nova nota apenas quando houver uma entidade, projeto, área ou conceito com valor próprio;
4. criar links entre notas relacionadas;
5. manter índices e contexto ativo atualizados;
6. evitar duplicação, ruído e crescimento descontrolado.

## O que a memória deve ser

- Uma wiki semântica persistente.
- Um mapa vivo do entendimento do James.
- Uma base de continuidade entre sessões.
- Um sistema de conhecimento legível em Obsidian.
- Um artefato versionado em Git.
- Uma camada de síntese acima do histórico bruto.

## O que a memória não deve ser

- Um log infinito.
- Um backup de conversas completas.
- Um depósito de arquivos sem síntese.
- Um banco vetorial.
- Uma coleção de notas vazias.
- Um substituto de SOUL.md, USER.md ou MEMORY.md.

## Relação com outras memórias

- SOUL.md define quem James é.
- USER.md define quem Lucas é.
- MEMORY.md mantém contexto ativo e ponteiros de curto prazo.
- Este Vault mantém conhecimento consolidado de longo prazo.
- Skills mantêm procedimentos reutilizáveis.
- state.db preserva episódios e histórico bruto.

## Regra de ouro

Antes de criar, procurar. Antes de registrar, sintetizar. Antes de expandir, perguntar se aquilo terá valor futuro.
